# Product Description

## User Problem

Product managers and operators often ask data questions in natural language, but generic AI assistants can produce confident answers without checking metric definitions, SQL scope, abnormal dates, or analyst review requirements.

## Target Users

DataPilot AI targets product managers, growth operators, lifecycle PMs, data analysts, and product analytics interns working in data-heavy SaaS or marketplace teams.

## Core Scenario

A PM asks, "Why did profit decline this month?" DataPilot AI identifies the intent, checks the Metric Registry, finds that profit is undefined, refuses to answer directly, suggests governed proxy metrics, and opens a metric definition request for analyst review.

## Product Value

The product makes AI data assistance more trustworthy by combining natural language UX with metric governance, transparent product-level trace, reusable Agent Skills, and human validation.

## MVP Scope

The first MVP includes polished routes, mock data, domain types, Agent Skill definitions, Metric Registry placeholders, analyst review story, ops dashboard placeholders, AI radar notes, PRD workspace, and case study framing.

## Differentiation

DataPilot AI is not a generic chatbot or dashboard. Its defining behavior is knowing when not to answer, then converting the refusal into a governance and product operations workflow.

## Risks And Fallbacks

Key risks include excessive refusal, low user trust in proxy metrics, weak intent mapping for informal language, and analyst review bottlenecks. Fallbacks include false rejection logging, confidence thresholds, metric request queues, and skill evaluation loops.

## Future Roadmap

Next milestones include the honest refusal assistant flow, Metric Registry approval loop, Daily Triage workflow, Ops dashboard with False Rejection Log, Analyst Review correction story, and detailed PRD and case study pages.
